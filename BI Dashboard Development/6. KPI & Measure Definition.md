# 📋 Step 6: KPI & Measure Definition

The goal of this phase is to **define, implement, and validate all KPIs and measures** so they are accurate, consistent, and aligned with business requirements.

## KPI & Measure Definition Questionnaire

### 📊 KPI Identification

* [ ] Which KPIs are required for the dashboard?
* [ ] Which KPIs are mandatory and which are optional?
* [ ] What business objective does each KPI support?
* [ ] Who owns or approves each KPI?

---

### 📝 KPI Definition

* [ ] What is the business definition of each KPI?
* [ ] How should users interpret the KPI?
* [ ] Are there any business assumptions?
* [ ] Is the KPI documented?

---

### 🧮 Measure Calculation

* [ ] What is the calculation formula?
* [ ] Which fields are used in the calculation?
* [ ] Are there any business rules or exceptions?
* [ ] Should the calculation be performed in ETL or in the BI tool?

---

### 🔍 Filters & Context

* [ ] Should the KPI respond to user selections?
* [ ] Should any filters be ignored?
* [ ] Are default filters required?
* [ ] Does the KPI require Set Analysis or filter context?

---

### 📅 Time Intelligence

* [ ] Does the KPI require Year-to-Date (YTD)?
* [ ] Month-to-Date (MTD)?
* [ ] Quarter-to-Date (QTD)?
* [ ] Previous Period comparison?
* [ ] Year-over-Year (YoY)?
* [ ] Month-over-Month (MoM)?
* [ ] Rolling periods (7, 30, 90, 365 days)?

---

### 🎯 Targets & Benchmarks

* [ ] Does the KPI have a target value?
* [ ] Is there a threshold for alerts?
* [ ] Should variance from target be calculated?
* [ ] Should achievement percentage be displayed?

---

### 📐 Formatting

* [ ] What format should the KPI use?

  * Number
  * Currency
  * Percentage
  * Decimal
  * Duration
* [ ] How many decimal places should be displayed?
* [ ] Should large values be abbreviated (K, M, B)?

---

### 🎨 Conditional Formatting

* [ ] Should colors indicate performance?
* [ ] What are the success, warning, and failure thresholds?
* [ ] Are icons or indicators required?
* [ ] Should trend arrows be displayed?

---

### ✔️ Validation

* [ ] Does the KPI match the business definition?
* [ ] Does the value match the source report?
* [ ] Has the calculation been tested with different filters?
* [ ] Have edge cases been validated (NULLs, zero values, negative values)?

---

### 📄 Documentation

* [ ] Is every KPI documented?
* [ ] Is the calculation formula documented?
* [ ] Are business rules documented?
* [ ] Are assumptions and limitations documented?

---

# ✅ Deliverables

By the end of this phase, you should have:

* [ ] KPI catalog
* [ ] Business definitions for all KPIs
* [ ] Measure formulas
* [ ] Time intelligence measures (YTD, MTD, YoY, etc.)
* [ ] Target and benchmark definitions
* [ ] Conditional formatting rules
* [ ] KPI validation report
* [ ] KPI documentation


---

> **Note:** Focus only on the **logic and accuracy** of KPIs in this phase. Do **not** decide where they will appear on the dashboard. Visual placement, layout, and user experience will be addressed in the next phase: **Dashboard Wireframing & Layout Design**.
