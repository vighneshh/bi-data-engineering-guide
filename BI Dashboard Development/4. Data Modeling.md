# 📋 Step 4: Data Modeling

The goal of this phase is to **design a scalable, efficient, and easy-to-maintain data model** that supports all business requirements and KPIs.

## Data Modeling Questionnaire

### 🏗️ Model Design

* [ ] What type of data model should be used? (Star Schema, Snowflake Schema, Flat Model)
* [ ] What is the central fact table?
* [ ] What are the dimension tables?
* [ ] Does the model support all business requirements?

---

### 📊 Fact Tables

* [ ] Which tables contain measurable business events?
* [ ] What is the grain (lowest level of detail) of each fact table?
* [ ] Are all required measures included?
* [ ] Are there duplicate fact records?

---

### 🗂️ Dimension Tables

* [ ] Which tables describe the business entities?
* [ ] Are dimensions descriptive and reusable?
* [ ] Are hierarchies defined?
* [ ] Do dimensions contain unique keys?

---

### 🔑 Keys & Relationships

* [ ] What is the primary key for each dimension?
* [ ] What is the foreign key in the fact table?
* [ ] Are relationships one-to-many where expected?
* [ ] Are there any many-to-many relationships?
* [ ] Are bridge tables required?

---

### 📅 Calendar / Date Dimension

* [ ] Is a dedicated calendar table required?
* [ ] Does it include Year, Quarter, Month, Week, Day?
* [ ] Does it support fiscal calendars?
* [ ] Are multiple date fields required (Order Date, Invoice Date, Due Date)?

---

### 📈 Measures & Calculations

* [ ] Which calculations should be performed in the data model?
* [ ] Which calculations should be left for the BI tool?
* [ ] Are derived columns required?
* [ ] Are aggregation levels clearly defined?

---

### 🏷️ Naming Standards

* [ ] Are table names meaningful and consistent?
* [ ] Are field names business-friendly?
* [ ] Are duplicate or ambiguous names avoided?
* [ ] Are naming conventions documented?

---

### 🔄 Performance Optimization

* [ ] Can unnecessary tables or columns be removed?
* [ ] Are unnecessary joins avoided?
* [ ] Can lookup tables be merged?
* [ ] Is the model optimized for fast querying?

---

### 🔍 Validation

* [ ] Do all KPIs map correctly to the model?
* [ ] Can every business question be answered?
* [ ] Are all relationships working correctly?
* [ ] Are there any circular references or synthetic keys (if applicable)?

---

### 📄 Documentation

* [ ] Is the data model diagram created?
* [ ] Are table relationships documented?
* [ ] Are assumptions documented?
* [ ] Are calculated fields documented?

---

# ✅ Deliverables

By the end of this phase, you should have:

* [ ] Final data model (Star/Snowflake Schema)
* [ ] Fact and dimension table design
* [ ] Relationship diagram
* [ ] Calendar/Date dimension
* [ ] Defined keys and relationships
* [ ] Naming convention document
* [ ] Calculation mapping
* [ ] Data model documentation


---

> **Note:** The output of this phase is the **blueprint** for development. No data transformation or loading should occur yet. The next phase, **ETL / Data Preparation**, will implement the model by extracting, transforming, and loading the data.
