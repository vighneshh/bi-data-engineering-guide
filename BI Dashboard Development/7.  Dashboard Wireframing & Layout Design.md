# 📋 Step 7: Dashboard Wireframing & Layout Design

The goal of this phase is to **design the dashboard layout and user experience** before development begins. Focus on **what users will see**, **how they will interact with the dashboard**, and **how information will be organized**.

## Dashboard Wireframing & Layout Design Questionnaire

### 🎯 Dashboard Purpose

* [ ] What is the primary purpose of this dashboard?
* [ ] Who is the target audience?
* [ ] What should users understand within the first 10 seconds?
* [ ] What are the most important KPIs to highlight?

---

### 📄 Dashboard Structure

* [ ] How many dashboard pages (sheets) are required?
* [ ] What is the purpose of each page?
* [ ] Should there be an Executive Summary page?
* [ ] Should there be Detail or Analysis pages?

---

### 📊 KPI Placement

* [ ] Which KPIs should appear at the top?
* [ ] Which KPIs should be grouped together?
* [ ] Which KPIs require comparison?
* [ ] Which KPIs need trend indicators?

---

### 📈 Visualization Selection

* [ ] What is the best visualization for each KPI?

  * KPI Card
  * Bar Chart
  * Line Chart
  * Combo Chart
  * Pie / Donut Chart
  * Pivot Table
  * Table
  * Scatter Plot
  * Map
  * Heat Map
  * Gauge
* [ ] Does each visualization clearly answer a business question?
* [ ] Can any visualization be simplified?

---

### 📐 Layout & Grid

* [ ] Is the dashboard organized using a consistent grid?
* [ ] Are related visuals grouped together?
* [ ] Is there enough spacing between objects?
* [ ] Is the dashboard balanced without empty areas?

---

### 🎛️ Filters & Navigation

* [ ] Which filters should be visible on every page?
* [ ] Where should filters be placed?
* [ ] Are filter names easy to understand?
* [ ] Are navigation buttons or tabs required?
* [ ] Is a Reset Filters button required?

---

### 🔍 User Interaction

* [ ] Should users be able to drill down?
* [ ] Are drill-through pages required?
* [ ] Should charts cross-filter each other?
* [ ] Are tooltips required?
* [ ] Should users be able to export data?

---

### 🎨 Visual Design

* [ ] Does the dashboard follow company branding?
* [ ] Are fonts consistent?
* [ ] Are colors used consistently?
* [ ] Is conditional formatting easy to understand?
* [ ] Is color used to highlight insights rather than decorate?

---

### 📱 Responsiveness & Accessibility

* [ ] Will the dashboard be viewed on desktop only?
* [ ] Is tablet or mobile support required?
* [ ] Are fonts readable on different screen sizes?
* [ ] Is the color contrast accessible?
* [ ] Can users easily interact with filters and charts?

---

### 📝 Labels & Titles

* [ ] Does every chart have a meaningful title?
* [ ] Are KPI names business-friendly?
* [ ] Are axis labels and legends clear?
* [ ] Are units (%, $, Count, Days, etc.) displayed correctly?

---

### 👀 User Experience (UX)

* [ ] Is the dashboard easy to navigate?
* [ ] Can users find important information quickly?
* [ ] Is unnecessary information removed?
* [ ] Does the dashboard follow a logical reading flow (Top → Bottom, Left → Right)?
* [ ] Does every visual provide value?

---

### 📄 Wireframe Review

* [ ] Has a low-fidelity wireframe been created?
* [ ] Has the layout been reviewed with stakeholders?
* [ ] Have stakeholder comments been incorporated?
* [ ] Is the layout approved before development?

---

# ✅ Deliverables

By the end of this phase, you should have:

* [ ] Dashboard wireframes (low-fidelity or high-fidelity)
* [ ] Dashboard page structure
* [ ] Visualization mapping
* [ ] KPI placement plan
* [ ] Filter and navigation design
* [ ] Layout grid
* [ ] UX and interaction design
* [ ] Approved dashboard mockup

---

# ✅ Exit Criteria

Move to the next stage only when all of the following are complete:

* [ ] Dashboard pages are finalized.
* [ ] Layout and grid are approved.
* [ ] KPI placement is finalized.
* [ ] Visualizations are selected.
* [ ] Filters and navigation are designed.
* [ ] User interactions are defined.
* [ ] Branding and visual design guidelines are applied.
* [ ] Dashboard wireframes are reviewed and approved by stakeholders.

---

> **Note:** This phase focuses on **design only**. Do **not** build any charts or write expressions yet. The objective is to create a clear blueprint for the dashboard so development is faster, more consistent, and requires minimal rework. The next phase is **Dashboard Development**, where the approved wireframe is implemented in the BI tool.
