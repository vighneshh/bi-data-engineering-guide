# 📋 BI Dashboard Development – Step 1: Business Requirements

The goal of this phase is to understand **what needs to be built** before thinking about data, design, or development.

## Business Requirement Questionnaire

### 🎯 Business Objective

* [ ] What business problem are we trying to solve?
* [ ] Why is this dashboard needed?
* [ ] What decisions should this dashboard support?

---

### 👥 Stakeholders

* [ ] Who requested the dashboard?
* [ ] Who will use the dashboard?
* [ ] What are their roles (Executive, Manager, Analyst, Operations, etc.)?

---

### ❓ Business Questions

* [ ] What key questions should the dashboard answer?
* [ ] What insights are users expecting?
* [ ] What actions should users take based on these insights?

---

### 📊 KPIs & Metrics

* [ ] Which KPIs should be displayed?
* [ ] How is each KPI calculated?
* [ ] Are there any business rules or target values?

---

### 🗂️ Dimensions

* [ ] Which dimensions should users analyze the KPIs by?

  * Date
  * Year / Quarter / Month / Week
  * Region
  * Country
  * Customer
  * Product
  * Department
  * ERP
  * Other: ___________

---

### 🎛️ Filters

* [ ] Which filters should be available?
* [ ] Which filters should be selected by default?
* [ ] Are any mandatory filters required?

---

### 🔍 Drill-down

* [ ] Do users need to drill down into the data?
* [ ] What should the drill path be?

  * Example: Region → Country → Customer

---

### 📈 Dashboard Expectations

* [ ] Which KPIs should be highlighted?
* [ ] Are there preferred chart types?
* [ ] Are there any dashboard mockups or reference reports?

---

### ✔️ Success Criteria

* [ ] How will users know the dashboard meets their requirements?
* [ ] What are the acceptance criteria for sign-off?
